# Corner-Grocer-Tracker

Project Summary & Problem Solved
For this project, I developed an item tracking program for the Corner Grocer. The store needed a way to analyze daily text records of purchased items, which are generated in chronological order throughout the day. The program processes this data to determine the frequency of each item purchased, allowing the store to identify trends and create the most effective layout for their produce section.

What I Did Well
I am particularly proud of how I implemented Maps in C++ to efficiently store and count the frequency of each grocery item. I also feel I did a great job formatting the text based histogram to ensure the data was visually clear and easy for the end user to read.

I could enhance my code by adding more robust user input validation and error handling. For example, if a user inputs a letter when the menu expects an integer, the program should catch the error rather than crashing or looping infinitely. Anticipating and responding to these runtime or user errors would make the application much more secure, stable, and user-friendly.

Challenges & Overcoming Them
The most challenging piece of code to write was properly implementing the file input and output operations, specifically ensuring the program correctly read the data from File.txt and formatted the backup data into frequency.dat without dropping any values. I overcame this by carefully reviewing the materials covered in zyBook on File I/O and utilizing C++ reference documentation to ensure my syntax was exact. These documentation sites are valuable resources I will keep in my support network.

Transferable Skills
Working with the Map data structure is highly transferable, as key-value pairing is a fundamental concept used across many programming languages (like dictionaries in Python). Additionally, applying Object Oriented Programming principles by creating dedicated classes with public and private sections will be essential for building larger, more complex software in the future.

Maintainability & Readability
I ensured the program was maintainable and readable by consistently applying industry standard best practices. I used clear, descriptive naming conventions for all my variables and functions so their purpose is immediately obvious. I also included in line comments throughout the source code to explain the logic behind complex blocks, making it adaptable for any developer who might need to update the code later.
